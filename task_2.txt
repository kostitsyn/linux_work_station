1. Самоподписанный сертификат - это сертификат, который создали мы сами, а не специализированной для этого структурой (ЦС - доверенный центр сертификации)

Создаём сертификат и закрытый ключ:

openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/ssl/private/example.com.key -out /etc/ssl/certs/example.com.pem
где
example.com.key - закрытый ключ
example.com.pem - публичный сертификат


Создаём сильную группу Диффи-Хеллмана
openssl dhparam -out /etc/ssl/certs/dhparam.pem 2048


2. Создаём файл конфигурации "self-signed.conf" и указываем путь до ключа и сертификата:

cat > /etc/nginx/snippets/self-signed.conf
 
ssl_certificate /etc/ssl/certs/example.com.pem;
ssl_certificate_key /etc/ssl/private/example.com.key;


Создаём файл конфигурации "ssl-params.conf" для поддержки параметров Strong и Secure SSL:

cat /etc/nginx/snippets/ssl-params.conf
ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
ssl_prefer_server_ciphers on;
ssl_ciphers "EECDH+AESGCM:EDH+AESGCM:AES256+EECDH:AES256+EDH";
ssl_ecdh_curve secp384r1;
ssl_session_cache shared:SSL:10m;
ssl_session_tickets off;
ssl_stapling off;
ssl_stapling_verify on;
resolver 8.8.8.8 8.8.4.4 valid=300s;
resolver_timeout 5s;
add_header Strict-Transport-Security "max-age=63072000; includeSubdomains";
add_header X-Frame-Options DENY;
add_header X-Content-Type-Options nosniff;
 
ssl_dhparam /etc/ssl/certs/dhparam.pem;


Редактируем файл конфигурации виртуального хоста: раскоментируем SSL протокол по 443 порту, добавляем ранее созданные файлы конфигурации:

server {
        listen 80 default_server;
        listen [::]:80 default_server;

        # SSL configuration

         listen 443 ssl default_server;
         listen [::]:443 ssl default_server;
        #
        # Note: You should disable gzip for SSL traffic.
        # See: https://bugs.debian.org/773332
        #
        # Read up on ssl_ciphers to ensure a secure configuration.
        # See: https://bugs.debian.org/765782
        #
        # Self signed certs generated by the ssl-cert package
        # Don't use them in a production server!
        #
         include snippets/self-signed.conf;
         include snippets/ssl-params.conf;

Проверяем на ошибки и перезапускаем:
nginx -t
systemctl restart nginx

3. В окне браузера видим сообщение: Warning: Potential Security Risk Ahead
Firefox detected a potential security threat and did not continue to localhost. 
If you visit this site, attackers could try to steal information like your passwords, emails, or credit card details

4. Ассиметричное шифрование сильно нагружает процессор. Симметричное шифрование более простое, быстрее выполняется.
